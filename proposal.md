# AI 视觉对话助手 — 需求提案 (Proposal)

## 项目概述

**What**: 浏览器端 AI 视觉对话应用 — 打开摄像头与麦克风，AI 实时看到画面、听懂语音，用语音给予回应。

**Why**: 七牛云「AI 视觉对话助手」赛题，探索端云协同的实时多模态交互。

**Target users**: 评委（演示场景：日常物品识别、环境问答、操作指引等）。

## 范围

**In scope**:
- 摄像头实时画面捕获 → AI 视觉理解
- 麦克风拾音 → 语音识别 (ASR)
- AI 理解画面+文本 → 生成回答 (VLM)
- 回答 → 语音合成播报 (TTS)
- 端云混合成本控制策略（端侧预处理 + 缓存 + 本地模型优先）
- 设计文档（用户故事、成本控制技巧）

**Out of scope**:
- 移动端原生 App（仅 Web）
- 用户登录/注册体系
- 后端数据库持久化
- 多人协作

## 功能需求

### P0 — 必须有 (MVP)

- [ ] **FR-001**: 摄像头实时画面捕获 — 浏览器调用摄像头，Canvas 截帧渲染到页面
- [ ] **FR-002**: 端侧帧去重 — 帧差算法检测画面变化，仅变化帧触发后续处理，静态场景不消耗 token
- [ ] **FR-003**: 麦克风拾音与 VAD — Web Audio API 录音，端侧静音检测 (VAD)，仅检测到语音时才触发 ASR
- [ ] **FR-004**: 语音转文字 (ASR) — Web Speech API 主力（免费），失败时 fallback 七牛云 ASR
- [ ] **FR-005**: 视觉理解 + 回答生成 (VLM) — Transformers.js 小模型端侧优先（免费），简单问题就地回答；复杂/失败时上传压缩帧到七牛云 API
- [ ] **FR-006**: 文字转语音 (TTS) — Web Speech Synthesis 主力（免费），失败时 fallback 七牛云 TTS
- [ ] **FR-007**: 答案缓存 — 同类画面+同类问题命中缓存则直接返回，不重复调 API
- [ ] **FR-008**: 端侧预处理管线 — 帧分辨率降低 (max 640px)、JPEG 压缩 (quality 0.6)，最小化上传数据量
- [ ] **FR-009**: 低功耗模式 — 无交互 10 秒后帧率降至 0.5fps，检测到语音/画面变化恢复
- [ ] **FR-010**: 交互 UI — 摄像头预览区 + 对话气泡 + 状态指示（监听中/思考中/播报中）

## 非功能需求

| 分类 | 要求 |
|------|------|
| 性能 | ASR 识别 < 500ms；VLM 端侧回答 < 2s，云端回答 < 5s；帧处理 30fps 不掉帧 |
| 成本 | 单次交互云端 token 消耗 < 500（经帧去重+压缩+缩略后） |
| 安全 | 摄像头/麦克风权限用户主动授权；API Key 不放前端，走后端代理 |
| 兼容性 | Chrome 90+, Edge 90+, Safari 15+（Web Speech API / getUserMedia 兼容） |
| 可用性 | 一键开始，无需配置；异常状态有提示（权限拒绝、网络断开、API 超时） |
| 可维护性 | TypeScript strict；ESLint + Prettier；每个模块可独立测试 |

## 技术约束

- **语言/框架**: TypeScript + Next.js (React)
- **端侧 AI**: Transformers.js (浏览器内推理)
- **云端 API**: 七牛云 ASR / VLM / TTS
- **部署**: Vercel / 静态导出，单命令 `npm run dev` 可运行
- **存储**: 无需持久化存储；缓存用内存 Map

## 端云协同策略（成本控制核心）

```
用户说话
  → VAD 静音检测（端侧，免费）→ 静音：丢弃，不触发任何处理
  → 有语音：Web Speech API ASR（端侧，免费）→ 成功：得到文本
  → 失败：七牛云 ASR（云端，付费，仅音频片段）
  → 文本 + 当前帧：答案缓存命中？（端侧，免费）→ 命中：直接返回
  → 未命中：本地小模型能答？（端侧 Transformers.js，免费）→ 能：返回
  → 不能：压缩帧 + 问题 → 七牛云 VLM（云端，付费，仅关键帧+低分辨率）
  → 回答：Web Speech Synthesis TTS（端侧，免费）→ 成功：播报
  → 失败：七牛云 TTS（云端，付费）
```
