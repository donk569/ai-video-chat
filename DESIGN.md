## 成本控制技巧

### 规划 vs 实际

| 层级 | 技术 | 策略 | 实测节省 |
|------|------|------|----------|
| L1: 低功耗 | requestAnimationFrame 节流 | 无交互 10s → 0.5fps，交互恢复 30fps | ~60x |
| L2: 帧差去重 | 像素对比 (R/G/B) | 变化 <15% 不处理 | ~10x |
| L3: 降分辨率 | Canvas resize | 640px max width | ~5x |
| L4: JPEG 压缩 | canvas.toDataURL('image/jpeg', 0.6) | 原始 → 压缩后 size < 20% | ~10x |
| L5: 答案缓存 | LRU Map (容量 100) | 同类问题 hash 命中 → 0 云端调用 | ~30-50% |
| L6: 本地模型 | Transformers.js | 浏览器内推理，0 云端调用 | ~20-30% |

### 成本追踪

每个模块在调用云端 API 时记录：
- `cloudCalls`: 云端调用次数
- `tokensUsed`: Token 消耗量
- `estimatedCostRMB`: 预估人民币成本（按 0.01 元/次 + 0.0001 元/token 估算）

调试面板实时显示累计成本，方便监控。

## 架构决策记录

### 1. 端侧优先的 fallback 链

**决策**: 所有 AI 能力（ASR/VLM/TTS）都先尝试端侧实现，失败后才上云。

**理由**:
- 端侧 0 成本（Web Speech API、Transformers.js 免费）
- 云端 API 按量计费，仅在必要时调用
- 网络延迟：端侧 < 100ms，云端 > 500ms

**权衡**: 端侧模型精度可能低于云端，但对于大部分常见场景已足够。

### 2. Next.js API Routes 代理云端 API

**决策**: 通过 Next.js 服务端代理七牛云 API 请求。

**理由**:
- API Key 存在于服务端环境变量，不暴露到前端 bundle
- 可以添加请求校验（如图片大小限制 500KB）
- 统一错误处理和日志

**权衡**: 增加了一次网络跳转（前端 → Next.js → 七牛云），增加 ~50ms 延迟。

### 3. 帧预处理优先于上传

**决策**: 所有帧在端侧完成降分辨率 + JPEG 压缩后再上传。

**理由**:
- 原始帧 1080p 约 2MB，压缩后 640px JPEG q=0.6 约 30KB
- 减少 ~60x 上传带宽
- 减少云端处理成本（小图片处理更快）

### 4. 答案缓存

**决策**: LRU 缓存（容量 100），key = question + frame hash。

**理由**:
- 同类场景同一物体，用户可能问类似问题
- 缓存命中完全避免云端调用
- LRU 保证内存可控

## 测试策略

- **单元测试**: 每个模块独立测试（Vitest），mock 浏览器 API
- **组件测试**: React Testing Library，测试交互行为
- **集成测试**: 全链路 mock 测试（摄像头 + 麦克风 → ASR → VLM → TTS）
- **覆盖率目标**: >70%

## 部署

```bash
pnpm build  # 生成生产构建
pnpm dev    # 开发模式一键启动 localhost:3000
```

支持部署到 Vercel、Netlify 等平台。环境变量需在平台配置。
