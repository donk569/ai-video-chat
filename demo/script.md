# Demo 演示稿

---

> **视频时长**：约 2 分钟  
> **语速**：中等偏慢，自然口语

## 开场（10 秒）

*[打开 Landing 页面]*

大家好。这是 AI 视觉对话助手——一个浏览器端的 AI 视觉对话应用。

打开摄像头，AI 就能看见你的世界；开口说话，AI 就能听懂你的声音。而且，我们做到了一件事：**极致低成本**。

---

## 落地页展示（15 秒）

*[滚动 Landing 页]*

这是产品首页。上面介绍了核心功能：语音对话、文字交流、实时画面、打断即停。

往下看，这是我们最核心的设计——**六层成本漏斗**。

从原始 30 帧每秒，经过低功耗降帧、帧差去重、分辨率压缩、JPEG 压缩、答案缓存，最后才是云端 API。

每一层都在省钱。不到万不得已，绝不花一分钱。

## 文字对话（15 秒）

*[打字输入]*

先试试打字。输入"你好，介绍一下你自己"。

可以看到状态指示器：灰色就绪，绿色聆听，蓝色思考，黄色播报。

AI 用语音回答我。同时，右上角的调试面板实时显示云端调用次数和成本。

---

## 语音对话（20 秒）

*[说话测试]*

现在试试语音。看到绿色的"正在聆听"提示，说明麦克风在工作。

[说话]"今天天气怎么样"

我一说完，AI 就开始思考，然后语音回答。整个过程没有调用七牛云的 ASR——用的是浏览器自带的 Web Speech API，端侧完成，零成本。

我还可以随时打断它。比如说[打断]"停，换一个话题"——AI 立刻闭嘴，开始听我说新的内容。

---

## 技术架构（15 秒）

*[切到 GitHub README 或代码截图]*

简单说一下技术实现。

前端用 Next.js 16，后端 API 路由代理七牛云的 VLM、ASR、TTS 接口。

AI 引擎是 DeepSeek-V3 和 Qwen-VL 视觉模型。

语音识别优先用浏览器原生 Web Speech API，失败才上云。

语音合成同理，Web Speech Synthesis 优先，七牛 TTS 做兜底。

整套架构的核心思想就是：**端侧优先，云端兜底**。

## 备用要点（如果评委提问）

| 问题 | 回答 |
|------|------|
| 成本能省多少？ | 六层漏斗理论压缩比超 30000 倍，实际使用中大部分请求在本地完成，单次对话成本趋近于零 |
| 支持哪些浏览器？ | Chrome 和 Edge（Chromium 内核），需要 Web Speech API 和 getUserMedia 支持 |
| 端侧模型是什么？ | Transformers.js 在浏览器内推理，当前版本默认未加载，互联网恢复后可启用 |
| 能同时处理多少人？ | 单用户浏览器端应用，服务端只需要代理 API，几乎无服务端负载 |
| 有哪些待完善的地方？ | 移动端适配、深色模式（已有暗色设计）、浏览器兼容性测试 |
